# 로그인 · 테스트 계정 · 권한 가이드

전자정부 표준프레임워크 공통컴포넌트 [일반로그인 가이드](https://www.egovframe.go.kr/wiki/doku.php?id=egovframework:com:v2:uat:일반로그인)의 **표준 샘플 계정**을 그대로 제공합니다. 초기 데이터(시드)에 포함되어 별도 설정 없이 로그인 테스트가 가능합니다.

## 1. 표준 샘플 계정

비밀번호는 모두 `rhdxhd12`(= `공통12`를 영문 자판으로 입력한 값)이며, **ID·비밀번호는 대소문자를 구분**합니다.

| 사용자 구분 | ID | 비밀번호 | 대상 테이블 | 권한 |
|------|-----|---------|------|------|
| 업무사용자(직원) | `TEST1` | `rhdxhd12` | `TB_EMPLYR_INFO` | 일반 사용자(ROLE_USER) |
| 업무사용자(직원) | `webmaster` | `rhdxhd12` | `TB_EMPLYR_INFO` | 일반 사용자(ROLE_USER) |
| 일반회원 | `USER` | `rhdxhd12` | `TB_GNRL_MBER` | 일반 사용자(ROLE_USER) |
| 기업회원 | `ENTERPRISE` | `rhdxhd12` | `TB_ENTRPRS_MBER` | 일반 사용자(ROLE_USER) |
| 관리자 | `admin` | `1` | `TB_EMPLYR_INFO` | 관리자(ROLE_ADMIN) |

> 참고: 위 4개 표준 계정 외에 데모용 계정(`user`/`1` 업무사용자, `user1`/`1` 일반회원, `bizapproved` 기업회원)도 시드에 포함되어 있습니다.

## 2. 로그인 방법

1. 로그인 화면(`/uat/uia/egovLoginUsr.do`)에서 **사용자 구분**(업무사용자/일반회원/기업회원)을 선택합니다.
2. ID·비밀번호를 입력하고 로그인합니다. 구분(`USR`/`GNR`/`ENT`)에 따라 조회 대상 회원 테이블이 분기됩니다(`EgovLoginUsr_SQL_*.xml`).
3. 로그인에 성공하면 세션에 인증 정보가 저장되고 메인 화면으로 이동합니다.

## 3. 비밀번호 암호화 방식

- 저장 해시 = `Base64(SHA-256(id_bytes ‖ password_bytes))` — `EgovFileScrty.encryptPassword(password, id)`
- ID를 salt로 사용하므로 동일 비밀번호라도 계정마다 해시가 다르며, **대소문자가 다르면 다른 해시**가 됩니다(`test1` ≠ `TEST1`).
- 본 프로젝트 시드의 해시값은 공식 배포 DML(`script/dml/**/com_DML_*.sql`)의 값과 동일합니다.

## 4. 권한(RBAC)

- `admin` 계정만 **관리자(ROLE_ADMIN)** 입니다. 관리자 전용 상단 메뉴(회원·권한 / 코드·메뉴 / 시스템관리)가 노출됩니다.
- 표준 샘플 계정(`TEST1`/`webmaster`/`USER`/`ENTERPRISE`)은 모두 **일반 사용자(ROLE_USER)** 로, 관리자 전용 메뉴가 노출되지 않습니다.
- 관리자 메뉴 노출은 `templates/fragments/nav.html`의 `sec:authorize="hasRole('ADMIN')"` 로 제어됩니다.

## 5. 헤더 표시 · 본인 정보수정(마이페이지)

- 로그인하면 헤더에 사용자 이름과 함께 **사용자 구분 배지**(업무사용자/일반회원/기업회원)가 표시됩니다.
- 로그인한 모든 사용자는 헤더의 **"내 정보수정"** 링크(`/uss/umt/mypage.do`)에서 본인 정보를 조회·수정하고 비밀번호를 변경할 수 있습니다.
- 마이페이지는 **세션 본인 기준으로만** 동작합니다. (비로그인 상태에서는 예제 미리보기용으로 회원 유형을 전환해 볼 수 있습니다.)

## 6. 관련 시드/데이터 위치

- 런타임(HSQL): `src/main/resources/db/shtdb.sql`
- 운영 6종 DBMS: `DATABASE/{postgresql,oracle,mysql,cubrid,altibase,tibero}/all_ccm_data_*.sql`
